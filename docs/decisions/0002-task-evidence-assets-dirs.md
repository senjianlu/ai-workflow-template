# 0002:每个任务目录固定增设 evidence/ 与 assets/ 子目录

- 日期:2026-07-11
- 背景:rawf 任务的过程产物里,除方案/实现/评审记录外,还常有验证证据
  (测试输出、截图)与引用素材(Design 稿、参考图)。此前无约定落点,散落
  在任务目录根或临时路径,难以随任务一起留存与回溯。
- 决定:在 `.ai/<date>/<slug>/` 下固定设两个子目录——`evidence/`(证据)与
  `assets/`(资源),由 rawf-plan 建任务目录时一并创建,空目录用 `.gitkeep`
  占位。**二者均随任务一起入库**(与 `.ai/` 作为"随任务提交的过程痕迹"一致),
  不新增 `.gitignore` 忽略项。
  - 被否掉的备选:证据 gitignore、仅资源入库。否因:证据是"实现忠于 plan"
    的可回溯凭据,与实现记录同价值,不入库则换机/协作时丢失。
  - 落盘时机约束:证据/素材须在**评审前**落盘。`.ai/` 计入 review.sh 指纹,
    评审进行中写入会使评审无效(见 CLAUDE.md 硬规则)。
- 影响:
  - rawf-plan 建目录逻辑、AGENTS.md 目录约定、plan 模板、rawf-implement 自测
    指引均已标注两目录用途;rawf-report 的 `git add 任务目录`天然覆盖二者。
  - 残留风险:证据/资源含大二进制会膨胀 git 历史。当前按"均入库"接受此代价;
    若日后膨胀明显,可另开任务改为 evidence gitignore,届时新增决策取代本条。
