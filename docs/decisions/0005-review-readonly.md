# 0005:评审取消副本,只读沙箱直审原仓库,测试核验改走证据打回协议

- 日期:2026-07-18
- 背景:实现层评审(review.sh)原以 `cp -Rp` 全量复制工作区到 /tmp
  一次性副本、`--sandbox workspace-write` 运行 Codex,为的是保留评审者
  运行受限单元测试的能力(0004 的 B 案)。该机制的代价被两件事证实过高:
  ① 下游 cscheap-frontend 2026-07-17 因内存/swap 耗尽两次整机失去响应,
  取证认定全量复制(实测单次 2.4G,逐轮重复)是工作流侧主要放大器;
  ② 同日"副本瘦身"尝试(rsync 排除 + 依赖软链,任务已废弃)引出新一层
  复杂度——实测确认 codex 沙箱默认允许写 /tmp 与 $TMPDIR、软链 `dep/..`
  父目录穿越使写穿面覆盖整个主工作区(主工作区位于沙箱可写根下时实际
  可写),需要探针、护栏、布局边界持续维护。同时的经验事实是:既往任务
  中评审者复跑与实现者自测从未出现结果分歧,分歧全部发生在**证据完备性**
  上,而证据完备性只读即可审。
- 决定:采纳并推翻 0004 曾否掉的备选 A 并加以强化——
  - review.sh 取消副本,`codex exec --sandbox read-only -C <原仓库>`
    直审(与 plan-review.sh 同构);raw 输出经 `--output-last-message`
    由 codex CLI 进程写入任务目录(CLI 层写盘不经沙箱,plan-review.sh
    已长期验证);
  - 评审者**不运行任何测试**(含单元测试),全部测试用例统一走
    **证据打回协议**:实现者预先把每条用例的完整原始输出落任务目录
    evidence/,评审者只核证据并交叉比对;证据缺失/不完整 → blocker 且
    必须开出补交清单,矛盾 → 虚报 blocker(权威定义在
    .ai-workflow/review-standards.md「评审动作边界」);
  - 完整性指纹(pre/post hash)保留为双保险:检测评审期间主工作区并发
    写入,兜底沙箱失效;
  - 被否掉的备选:副本瘦身(rsync + 软链,复杂度见背景②)、`cp -al`
    硬链副本(写穿面更大)、维持现状(资源代价已酿成事故)。
- 影响:
  - 评审的复制开销与内存/IO 峰值归零,后台评审对开发机资源无叠加冲击;
  - review.sh 显著简化并与 plan-review.sh 同构(锁、轮次、指纹、交叉
    校验、原子发布、退出码语义均保持);
  - 抓虚报测试从"重跑比对"改为"证据审查",evidence/ 纪律成为承重墙:
    rawf-implement 第 5 步升级为所有用例完整原始输出强制落盘;
  - 残留风险:开发侧证据造假无法被复跑戳穿,依赖模型与流程纪律;若日后
    出现分歧或造假案例,新增决策引入抽查机制(如随机重跑某条用例);
  - 0004 整体被本条取代(其动作边界随 workspace-write 副本一并退役,
    新的动作边界见 review-standards.md 现行文本)。
